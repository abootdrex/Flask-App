  • 1. Abstraction In this internal implementation and complex logic is completely hidden from the user. In other words it is used to create a boundary between the application and the client programs. Moverover Objects are the building blocks of Object-Oriented Programming. An object contains some properties and methods we can hide them from the outer world through access modifiers. We can provide access only for required functions and properties to the other programs. This is the general procedure to implement abstraction in OOPS.
    1. Ex. Car car is a great example of abstraction.
      You can start a car by turning the key or pressing the
      start button. we do not need to know how the engine is getting started
      what all components your car has.
 



There are two types of abstraction

1.Data Abstraction
2.Process Abstraction


1.Data Abstraction When the object data is not visible to the outer world it creates data abstraction. If needed access to the Objects data is provided through some methods.


2. Process Abstraction We do not need to provide details about all the functions of an object. When we hide the internal implementation of the different functions involved in a user operation, it creates process abstraction.
